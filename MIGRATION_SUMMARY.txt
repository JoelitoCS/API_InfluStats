🔍 COMPARAR CÓDIGO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

BUSCAR USUARIO

MongoDB (ANTES):
  const user = await User.findOne({ email: emailLowercase });

PostgreSQL (AHORA):
  const user = await prisma.user.findUnique({
    where: { email: emailLowercase }
  });


CREAR USUARIO

MongoDB (ANTES):
  const usuario = await User.create({
    email, password
  });

PostgreSQL (AHORA):
  const usuario = await prisma.user.create({
    data: { email, password }
  });


COMPARAR CONTRASEÑA

MongoDB (ANTES):
  const esValida = await usuario.matchPassword(password);

PostgreSQL (AHORA):
  const esValida = await bcrypt.compare(password, usuario.password);

❓ PREGUNTAS FRECUENTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

P: ¿Por qué migrar de MongoDB a PostgreSQL?
R: PostgreSQL es más robusto, mejor para relaciones, SQL estándar.

P: ¿Qué es Prisma?
R: ORM que facilita trabajar con BD, genera queries automáticamente.

P: ¿Es gratis Supabase?
R: Sí, tier gratuito es generoso para desarrollo.

P: ¿Debo instalar PostgreSQL?
R: No, Supabase lo provee. Opcional si quieres local.

P: ¿Cambió mi API?
R: No, endpoints y responses son idénticos.

P: ¿Cambió el frontend?
R: No, funciona igual que antes.

P: ¿Qué pasa con mis datos en MongoDB?
R: Necesitas migrarlos manualmente (Ver MIGRATION_*).

P: ¿Es más rápido PostgreSQL?
R: Sí, especialmente con relaciones complejas.
